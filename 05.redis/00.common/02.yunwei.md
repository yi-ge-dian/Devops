3. 排查内存是否到达了设置的最大内存
```shell
redis-cli INFO memory | grep -E "used_memory:|maxmemory:"

解决方案
# 将最大内存临时调整为 8GB
redis-cli CONFIG SET maxmemory 8gb
# 生产环境中，Redis内存通常设置为物理机总内存的50%-75%（如16GB物理机可设置8-12GB）。这一范围兼顾了Redis自身内存开销（如数据结构、缓冲区）与其他系统服务（如操作系统、后台进程）的需求。

# 将淘汰策略临时调整为 allkeys-lru
redis-cli CONFIG SET maxmemory-policy allkeys-lru

# noeviction（默认）：
# 不淘汰任何key，新写入操作返回错误（OOM command not allowed）。仅适用于数据绝对不能丢失的场景（如实时交易系统），但需严格监控内存使用。
# allkeys-lru：
# 从所有key中淘汰最近最少使用（LRU）的key。适用于缓存场景（如热点数据缓存），能有效保留高频访问数据，提升缓存命中率。
# volatile-lru：
# 仅从设置了过期时间的key中淘汰LRU key。适用于部分数据需定期过期的场景（如会话缓存、临时数据），避免误删长期有效数据。

redis-cli --bigkeys # 查看内存占用较大的key
MEMORY USAGE key # 查看指定key的内存占用
```

4. 使用规范:

set user:id:01:keyname value

```text
# 第一次扫描，从 0 开始，匹配 pattern，每次建议扫描 100 个 key
SCAN 0 MATCH key* COUNT 100
# Redis 返回：下一个游标 "12345" 和 本次找到的 key 列表 ["key1", "key2"]

# 使用返回的新游标继续扫描，直到游标为 "0" 表示完成
SCAN 12345 MATCH key* COUNT 100
```

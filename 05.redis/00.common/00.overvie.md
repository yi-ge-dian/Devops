# redis

## 是什么
- redis 是一个开源的，基于内存的数据结构存储系统，可以用作数据库、缓存和消息中间件
- redis 支持多种类型的数据结构，如字符串（strings）、散列（hashes）、列表（lists）、集合（sets）、有序集合（sorted sets）等
- redis 支持持久化，可以将数据存储到磁盘上，以便在系统重启后恢复数据
- redis 支持多种编程语言，如 C、C++、Java、Python、PHP、Node.js 等

## 特点
- 高性能：redis 是基于内存的数据存储系统，因此读写速度非常快
- 数据结构丰富：redis 支持多种数据结构，如字符串、散列、列表、集合、有序集合等，可以满足各种应用场景的需求
- 持久化：redis 支持将数据存储到磁盘上，以便在系统重启后恢复数据
- 支持多种编程语言：redis 支持多种编程语言，如 C、C++、Java、Python、PHP、Node.js 等，可以方便地与其他系统进行集成

## 应用场景
- 缓存：redis 可以用作缓存系统，将频繁访问的数据存储在内存中，以提高系统的响应速度
- 数据库：redis 可以用作数据库，存储结构化数据，如用户信息、订单信息等
- 消息队列：redis 可以用作消息队列，实现发布/订阅模式，实现异步处理

## 基本命令
- set key value：设置键值对
- get key：获取键对应的值
- del key：删除键值对
- keys pattern：查找符合模式的键
- expire key seconds：设置键的过期时间
- ttl key：获取键的剩余过期时间
- flushdb：清空当前数据库
- flushall：清空所有数据库

## 数据结构
- 字符串（strings）：用于存储简单的键值对
- 散列（hashes）：用于存储键值对的集合，每个键对应一个散列
- 列表（lists）：用于存储有序的元素集合，可以添加、删除、获取元素
- 集合（sets）：用于存储无序的元素集合，可以添加、删除、获取元素
- 有序集合（sorted sets）：用于存储有序的元素集合，每个元素都有一个分数，可以添加、删除、获取元素
- 流对象

## 持久化
- RDB：将数据快照存储到磁盘上，可以在系统重启后恢复数据
- AOF：将每次写操作记录到日志文件中，可以在系统重启后恢复数据

## 高可用
- 主从复制：将数据复制到多个节点上，提高系统的可用性和容错能力
- 哨兵模式：监控主节点的状态，在主节点故障时自动切换到从节点
- 集群模式：将数据分散到多个节点上，提高系统的扩展性和性能
